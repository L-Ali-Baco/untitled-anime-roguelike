### Task 3: Implement Procedural Rolling Animation

**Files:**

- Modify: `src/server/Enemies/Keg.luau`

**Step 1: Add Tick override to Keg.luau to handle rolling**

We need to override `Tick` to call `BaseEnemy.Tick` and then apply visual rotation to `KegMesh`.
Add `self.KegMesh = model:FindFirstChild("KegMesh")` or fallback to iterating descendants to find a `MeshPart` if not found in the constructor `Keg.new`.

```luau
-- Add to Keg.new:
	self.KegMesh = model:FindFirstChild("KegMesh") or model:FindFirstChildWhichIsA("MeshPart")
```

```luau
-- Add to Keg.luau:
-- ============================================================
-- OVERRIDE: Tick
-- ============================================================
function Keg:Tick(dt: number)
	-- Capture old position before base tick
	local oldPos = self.RootPart and self.RootPart.Position

	-- Run base logic (movement, state transitions)
	BaseEnemy.Tick(self, dt)

	-- If dead, destroyed, or no RootPart/Mesh, skip rolling
	if self._destroyed or self.State == "Dead" or not self.RootPart or not self.KegMesh then
		return
	end

	-- Procedural Rolling
	if self.State == AIState.Chase and self.IsMoving then
		local newPos = self.RootPart.Position
		if oldPos then
			local distMoved = (newPos - oldPos).Magnitude
			if distMoved > 0 then
				-- Approximate radius from the mesh's Y size
				local radius = self.KegMesh.Size.Y / 2
				-- Calculate rotation angle (弧度 = 弧长 / 半径)
				local angle = distMoved / radius
				-- Rotate the mesh forward around its local X axis
				self.KegMesh.CFrame = self.KegMesh.CFrame * CFrame.Angles(angle, 0, 0)
			end
		end
	end
end
```

**Step 2: Ensure rolling logic doesn't override arming flash**

The arming flash logic uses `TweenService`, which might interact weirdly if we manually change `CFrame`. Wait, `CFrame` is separate from `Color` tweening.
Because we only roll when `self.State == AIState.Chase`, it naturally stops rolling when it enters `ARMING_STATE`.
